LeaseGuard

Cryptographic Impact-Bounded Authorization Under Disconnected Hybrid Infrastructure Control Planes

Offline-capable credentials — self-contained OAuth tokens, macaroons, cached workload identities, heartbeat- or epoch-revocable tokens — all bound time or delegation structure. None of them bound cumulative operational damage. LeaseGuard is a cryptographic lease that carries a multi-dimensional impact budget — action count, data volume, object count, financial ceiling, destructive-operation ceiling — enforced against tamper-evident local state, so a stale or revoked-but-unreachable credential can only ever do so much damage, no matter how long the control plane stays partitioned.

The gap

Time-bounded authorization doesn't bound damage

Ten closest systems were surveyed — ACE-OAuth, macaroons, UCON, SPIFFE/SPIRE, CP-ABE revocation, DPoP, CAEP, epoch-revocable pseudonyms, and two 2026 preprints on agent admission control and offline capability graphs. None combine a cryptographically enforced, multi-dimensional cumulative-impact ceiling with local, tamper-evident consumption state.

What every prior scheme bounds

A time window, an epoch, a heartbeat interval, or a delegation chain. Once issued, the credential's remaining authority is whatever is left of that window — not a function of how much has actually been done with it.

What LeaseGuard bounds

Cumulative operational impact — action count, data volume, object count, financial estimate, destructive-op count — tracked against a hardware-sealed local counter, independent of how long the disconnection lasts.

How it works

Issuance, enforcement, reconciliation

The enforcement point never needs to reach the authority to make a decision — it only needs to reach it eventually, to reconcile.

1. Issuance (connected)Authority evaluates policy, signs a lease scoped to one subject/audience/epoch with a bounded impact budget.
→
2. Local enforcement (any connectivity)Enforcement point checks the sealed local counter against every budget dimension before each operation. Reads always remain available.
3. Emergency procedureA destructive operation beyond its ceiling requires local dual-control, never a remote approval that can't arrive during a partition.
→
4. Reconciliation on reconnectAuthority verifies counter integrity, resolves forked replicas conservatively (sum, not max), issues the next epoch.

Time-bounded schemes grow roughly linearly with partition duration once revoked mid-partition — nothing locally bounds them. LeaseGuard plateaus at its authorized budget and stays there, at the cost of dropping to 60.2% availability (read-only) once the 24-hour run exhausts its financial-impact ceiling. The epoch-revocable and heartbeat baselines also stay flatter, but only by giving up far more availability — see the full CSV in the repo for every scheme × duration combination.

Stated honestly

Limitations

  • Local counter rollback is detectable at reconciliation, not cryptographically impossible, without a genuine hardware root of trust (TPM/TEE) — many real storage arrays and hypervisor hosts don't expose one today.
  • Financial-impact and destructive-op weights are policy-authored estimates, not independently verified real-world damage figures.
  • This is a simulator. Production integration with real Kubernetes admission webhooks, VMware/Hyper-V adapters, and storage-array APIs is future work.
  • Emergency-deny-witness suppression over a fully disconnected out-of-band channel is an open problem, not solved by this design.
